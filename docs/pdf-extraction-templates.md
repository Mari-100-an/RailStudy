# PDF 정제용 프롬프트 템플릿

이 문서는 PDF 자료를 학습 플랫폼에 맞게 정제하기 위한 프롬프트 템플릿입니다.

---

## 📚 과목 코드 참조

| 과목명 | subject 코드 |
|--------|-------------|
| 철도관련법 | `law` |
| 도시철도시스템 일반 | `urban` |
| 전기동차 구조 및 기능 | `emu` |
| 운전이론 일반 | `theory` |
| 비상시 조치 | `emergency` |

---

## 템플릿 1: 이론/개념 정리용

아래 프롬프트를 복사하여 외부 AI(Claude, ChatGPT 등)에 PDF와 함께 전달하세요.

```
당신은 철도 기관사 시험 학습 자료를 정리하는 전문가입니다.
첨부된 PDF를 분석하여 아래 JSON 형식으로 변환해주세요.

## 출력 형식

{
  "subject": "law",  // 과목코드: law, urban, emu, theory, emergency 중 하나
  "subjectName": "철도관련법",
  "chapters": [
    {
      "id": "law_ch1",  // 형식: {subject}_ch{번호}
      "name": "철도안전법",
      "order": 1,
      "sections": [
        {
          "id": "law_ch1_s1",  // 형식: {chapter_id}_s{번호}
          "title": "철도안전법의 목적과 정의",
          "content": "마크다운 형식의 본문 내용",
          "keyTerms": ["핵심용어1", "핵심용어2"],
          "examTips": "(선택사항)"
        }
      ]
    }
  ]
}

## content 작성 규칙

1. **마크다운 형식** 사용:
   - ## 대제목, ### 소제목
   - **굵게** 로 핵심 키워드 강조
   - 표는 마크다운 테이블 형식
   - > 인용문으로 포인트 강조

2. **구조화**:
   - 핵심 개념을 명확하게 정의
   - 관련 법령 조항 명시
   - 수치/기준은 정확하게 기재

3. **예시 content**:
```markdown
## 철도안전법의 목적 (제1조)

철도안전법은 **철도안전을 확보하기 위하여 필요한 사항을 규정**하고 **철도안전 관리체계를 확립**함으로써 **공공복리의 증진**에 이바지함을 목적으로 한다.

### 핵심 키워드
- 철도안전 확보
- 안전 관리체계 확립
- 공공복리 증진

| 용어 | 정의 |
|------|------|
| **철도** | 여객 또는 화물을 운송하는 데 필요한 철도시설과 철도차량... |

> ⚠️ 중요 포인트: "5년마다" 수립하는 것이 핵심!
```

## 주의사항
- 원문의 의미를 훼손하지 말 것
- 법령 조항 번호는 정확하게 기재
- 수치(기간, 금액, 거리 등)는 반드시 원문 그대로
- 너무 길면 여러 section으로 분리
```

## 템플릿 2: 문제 추출용

```
당신은 철도 기관사 시험 문제를 정리하는 전문가입니다.
첨부된 PDF에서 문제를 추출하여 아래 JSON 형식으로 변환해주세요.

## 출력 형식

{
  "questions": [
    {
      "id": "law_101",  // 형식: {subject}_{3자리번호}, 기존 문제와 겹치지 않게 100번대부터 시작
      "subject": "law",  // 과목코드: law, urban, emu, theory, emergency
      "chapter": "철도안전법",  // 해당 단원명
      "question": "철도안전법의 목적으로 가장 적절한 것은?",
      "options": [
        "철도 산업의 경쟁력 향상",
        "철도안전을 확보하기 위해 필요한 사항을 규정하고 철도안전 관리체계를 확립함으로써 공공복리의 증진에 이바지",
        "철도의 건설 및 유지보수",
        "철도종사자의 복지 향상"
      ],
      "answer": 1,  // 0부터 시작하는 인덱스 (0=첫번째, 1=두번째, 2=세번째, 3=네번째)
      "explanation": "철도안전법 제1조(목적): 이 법은 철도안전을 확보하기 위하여...",
      "source": "철도안전법 제1조"  // 출처 (법령, 교재 페이지 등)
    }
  ]
}

## 과목 코드 참조
- law: 철도관련법
- urban: 도시철도시스템 일반
- emu: 전기동차 구조 및 기능
- theory: 운전이론 일반
- emergency: 비상시 조치

## 주의사항
1. **options는 반드시 4개** (보기가 5개면 4개로 조정, 3개면 오답 1개 추가)
2. **answer는 0-indexed** (첫 번째 보기가 정답이면 0, 두 번째면 1...)
3. **explanation에 근거 명시** (법령 조항, 교재 출처 등)
4. **id는 과목별로 고유하게** (law_101, law_102, ... / urban_101, urban_102, ...)
5. 문제가 불명확하거나 오류가 있으면 수정하여 출력

## 기존 문제 ID 범위 (중복 방지)
- law: law_001 ~ law_011 사용 중 → law_100부터 시작
- urban: urban_001 ~ urban_010 사용 중 → urban_100부터 시작
- emu: emu_001 ~ emu_014 사용 중 → emu_100부터 시작
- theory: theory_001 ~ theory_013 사용 중 → theory_100부터 시작
- emergency: emergency_001 ~ emergency_008 사용 중 → emergency_100부터 시작
```

## 템플릿 3: 요약정리본 → 개념 연결용

```
당신은 철도 기관사 시험 학습 자료를 정리하는 전문가입니다.
첨부된 요약정리 PDF를 분석하여, 기존 단원 구조에 맞게 핵심 내용을 추출해주세요.

## 기존 단원 구조

### 1과목: 철도관련법 (law)
- 1장: 철도안전법
- 2장: 도시철도운전규칙
- 3장: 철도차량운전규칙

### 2과목: 도시철도시스템 일반 (urban)
- 1장: 도시철도 총론
- 2장: 철도안전
- 3장: 도시철도 차량일반
- 4장: 철도전기일반
- 5장: 전력설비일반
- 6장: 신호일반
- 7장: 전기설비일반
- 8장: 토목일반
- 9장: 영업일반
- 10장: 관제일반
- 11장: 운전취급일반

### 3과목: 전기동차 구조 및 기능 (emu)
- 1장: 전기동차 일반
- 2장: 특고압회로
- 3장: 견인장치
- 4장: 고압보조장치
- 5장: 저압보조장치
- 6장: 제동장치
- 7장: 운전보안장치

### 4과목: 운전이론 일반 (theory)
- 1장: 개요
- 2장: 기초공학이론
- 3장: 전기기기
- 4장: 운전성능
- 5장: 열차저항
- 6장: 선로공학
- 7장: 소음과 진동
- 8장: 운전계획
- 9장: 운전취급

### 5과목: 비상시 조치 (emergency)
- 1장: 인적오류
- 2장: 이례상황
- 3장: 고장조치

## 출력 형식

각 단원별로 아래 형식으로 출력:

---
## [과목명] - [단원명]

### 핵심 요약
(불릿 포인트로 핵심 내용 5-10개)

### 암기 포인트
(시험에 나올 수치, 기간, 조건 등)

### 비교 정리
(헷갈리기 쉬운 개념 비교 표)
---

이 내용은 나중에 기존 개념정리에 병합됩니다.
```

---

## 사용 순서

1. **표준교재 PDF** → 템플릿 1로 정제 → concepts.js에 추가
2. **요약정리본 PDF** → 템플릿 3으로 정제 → 기존 concepts에 보충
3. **기출/예상문제 PDF** → 템플릿 2로 정제 → questions.js에 추가

---

## 정제된 결과물 전달 방법

정제된 JSON을 저(GitHub Copilot)에게 전달할 때:

```
아래 JSON을 concepts.js / questions.js에 추가해줘:

{정제된 JSON 붙여넣기}
```

한 번에 너무 많으면 과목별/단원별로 나눠서 전달해주세요.
